# Plan 03-01: PoP File Upload

## Objective

Commit and verify the Proof of Performance (PoP) file upload feature. This allows Business users to attach actual files (PDF, PNG, JPG, DOCX up to 5 MB) when submitting PoP for an approved engagement, instead of only entering a document reference URL.

All implementation is already complete in the working tree as uncommitted changes.

## Context

- Phase 2 added the PoP attachment step: after Compliance approves, Business attaches a PoP URL to move the engagement to `pop_submitted`.
- Phase 3 extends this with file upload: files are stored locally under `uploads/pop/` and served via an authenticated API route.
- The `uploads/` directory is excluded from git via `.gitignore`.
- Middleware already passes all `/api/` routes through; the new endpoints self-authenticate via `auth()`.

## Tasks

### Task 1: Commit .gitignore update
**What:** Stage and commit the `.gitignore` change that excludes `uploads/`.
**Why:** Without this, uploaded PoP files would be tracked by git.
**Verification:** `git diff --name-only HEAD -- .gitignore` returns no output after commit.

### Task 2: Commit PoP upload and serve API routes
**What:** Stage and commit `app/api/engagements/pop-upload/route.ts` and `app/api/engagements/pop-file/[filename]/route.ts`.
**Why:** These implement the file upload endpoint (POST, saves to `uploads/pop/`) and serve endpoint (GET, auth-gated, path-traversal protected).
**Verification:** Files exist and TypeScript compiles clean.

### Task 3: Commit ActionPanel file upload UI
**What:** Stage and commit `components/engagement/ActionPanel.tsx`.
**Why:** Adds file upload button ("Choose file"), upload progress states, and OR divider alongside the existing manual URL input in the "Attach Proof of Performance" panel.
**Verification:** `npx tsc --noEmit` passes.

### Task 4: Commit engagement detail page PoP link rendering
**What:** Stage and commit `app/(app)/engagements/[id]/page.tsx`.
**Why:** Renders `/api/engagements/pop-file/` URLs as clickable "View attached file ↗" links rather than raw URL text.
**Verification:** TypeScript compiles. All existing tests pass (`npx jest --no-coverage`).

## Must-Haves

- [ ] All 4 files committed individually with descriptive messages
- [ ] TypeScript compiles clean after all commits
- [ ] All existing tests pass (no regressions)
- [ ] `uploads/` in .gitignore
- [ ] PoP upload API: auth guard, 5 MB limit, allowlist of MIME types, path-traversal protection on serve route
